Final Year Engineering Project · Team of 5

AutoTester 2.0

Automated student code evaluation framework
🐙 View on GitHub

AutoTester 2.0 (feat3p) is a Python-based CI/CD framework developed during the final year at ENSEEIHT to automate the evaluation of student assignments. Professors configure test scenarios in a config.py file; when a student commits their work, the pipeline automatically triggers, runs the defined test scenarios, and pushes a detailed feedback report back to the student's repository — all without manual intervention.

⚡
On-Demand Evaluation
Students request specific test scenarios via a modalites.txt file. The system enforces attempt limits and cooldown delays per scenario.
🔁
GitLab CI/CD Integration
Each student commit triggers a cross-project pipeline using GitLab trigger tokens, automatically launching the AutoTester evaluation pipeline.
🐋
Dockerized Deployment
A dedicated Docker image bundles all framework dependencies, enabling consistent and portable execution across any GitLab CI runner.
📋
Configurable Test Scenarios
Professors define scenarios with custom tools (Java compiler, blackbox, JUnit...), visibility flags, penalty weights, and attempt quotas via a Python config file.
